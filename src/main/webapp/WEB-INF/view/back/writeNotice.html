jingdaiblog 里一些规范（类名，方法名那些java规范当做常识就没有写了）

  1. mybatis的 XxxMapper 映射文件都包含一个 BaseResultMap 的 resultMap 用作结果集映射。
  2. POJO类的布尔属性不能加 is ，而数据库字段必须加 is_ 。这点参考阿里Java开发手册，POJO的布尔型变量加 is 会导致部分框架解析出现序列化错误。
  3. 项目中Service和Dao层方法一些规范：
    • 获取单个对象用 get 做前缀
    • 获取多个对象用 list 做前缀，对象名后面加 s
    • 插入用 insert 做前缀
    • 删除用 delete 做前缀
    • 表示用什么取得/操作对象使用 byXx ,例如 getUserById()
  4. 枚举类都要带上 Enum 后缀，枚举成员名称全部大写，单词下划线隔开。（枚举类是特殊的常量类）
  5. 包名统一使用小写，点分隔符之间有且仅有一个自然语义的英语单词，包名统一使用单数形式。
  6. 后面有了继续写。。。
